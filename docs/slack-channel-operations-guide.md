## 4. 매주 수요일 = 공유의 날 (운영 디테일)

3기는 매주 수요일이 어떤 형태로든 "공유의 날"로 고정됩니다. 빠지는 주가 없어야 전체 가시성이 유지됩니다.

### 4-1. 오프라인 공유회 주차

| 시점 | 액션 |
| --- | --- |
| 월 오전 | 채널에 §6-A 안내 발송 (회의실, 시간, 준비물) |
| 수 D-3시간 | 회의실 세팅 시작 |
| 수 D-1시간 | 채널에 "1시간 후 시작" 리마인드 + 위치 멘션 |
| 수 진행 | 운영진이 첫 발표자로 분위기 만들기, 한 명당 3~5분 발표 + 1~2분 Q&A |
| 수 종료 직후 | 사진 1~2장 + 인상적이었던 발표 1~2건 요약을 채널에 업로드 |

**진행 원칙:**
- "뭘 만들었나"보다 **"뭘 시도했고 뭐가 안 됐나"** 공유 유도
- 운영진이 첫 번째 발표자로 분위기 형성 (실패 경험 먼저 꺼내면 더 좋음)

### 4-2. 슬랙 스크럼 주차

오프라인 공유회가 없는 주에 진행. **20명 전원 필수.**

| 시점 | 액션 |
| --- | --- |
| 월 오전 | §6-B 안내 발송 (양식 첨부, 마감 시간 명시) |
| 수 10:00 | 운영진이 먼저 양식 작성해서 채널에 게시 (분위기 선도) |
| 수 15:00 | 미제출자 명단 작성 → 개별 DM 리마인드 (§6-D) |
| 수 18:00 | 마감. 미제출자는 별도 트래킹 시트에 기록 |

## 5. 슬랙 스크럼 운영 상세

### 5-1. 운영 원칙

1. **전원 필수** — 예외 없음. 못 하는 주차는 사전에 운영진에게 양해 메시지.
2. **운영진이 먼저** — 매번 운영진이 가장 먼저 작성. 늦게 올리면 모두가 늦어짐.
3. **막힌 것은 운영진이 즉시 연결** — "🚧 막힌 것"에 내용이 있으면 도울 수 있는 사람 즉시 태그.

### 5-2. 표준 양식

```
📌 위클리 스크럼 — [이름]
✅ 지난 2주간 한 것:
🎯 다음 2주간 할 것:
🚧 막힌 것 / 도움 필요:
```

### 5-3. 미제출자 관리

| 미제출 횟수 | 대응 | 자동화 |
| --- | --- | --- |
| **1회차** | 당일 15시 **공개 채널에 미제출자 멘션 + 가벼운 안내** (§6-D) | A3 자동 발송 |
| **2회 연속** | 마감 다음날 가벼운 DM ("요즘 어떠세요?") | 운영진 수동 (A8 알림 받음) |
| **3회 이상 연속** | 1:1 미팅 제안 (§6-I) | 운영진 수동 |

> 💡 대부분 "바빠서"가 아니라 **"뭘 해야 할지 몰라서"** 멈춰 있는 경우입니다. 비난조 금물, 도움을 주는 톤으로 접근.
>
> 💡 **3기 정책 변경:** 1회차 리마인드는 개별 DM 이 아닌 **공개 채널 멘션 안내**입니다. "다같이 하는 분위기" 형성이 목표이며, 이름만 멘션하고 톤은 가볍게 합니다.

| 장치 | 채널 액션 | 빈도 |
| --- | --- | --- |
| **이번 주 MVP 언급** | 오프라인 공유회 후 채널에 "오늘의 인상 깊은 사례" 1~2건 콕 집어 칭찬 | 격주 |
| **ai-줍줍 사례 공유** | 참가자 사례를 전사 채널로 끌어올려 인정 (§6-G) | 주 1회 |
| **비개발자의 바이브코딩 칭찬** | 비개발자가 뭐든 만들었을 때 운영진이 적극적으로 채널에서 환영 | 발생 시 |
| **첫 프로젝트 셀러브레이션** | 누구나 첫 프로젝트 완료 시 이모지 폭격 + 한 줄 코멘트 | 발생 시 |

**금지 사항:**
- ❌ 진행 속도 비교 ("A님은 벌써 3번째인데...") — 역효과
- ❌ 운영진이 직접 코딩해주거나 대신 만들어주기 — "방법 조언 O, 대신 해결 X"
- ❌ 미제출자 공개 비난 — DM으로만 처리

## 8. 이탈자 관리 (채널 관점)

이탈은 자연스러운 일. 2기에서도 37명 중 2명 이탈. 압박이 아닌 도움으로 접근.

### 이탈 징후 (채널에서 감지)

- 슬랙 스크럼 **2회 연속 미제출**
- 오프라인 모임 **2회 연속 불참**
- 프로젝트 페이지 업데이트 정체

### 대응 프로세스

1. 1:1 DM (§6-I) → 상황 확인
2. 업무 과부하 → 프로젝트 범위 축소 제안 ("작은 것 하나만이라도")
3. 의지 없음 → 부담 없이 중단해도 된다고 안내
4. 공식 이탈 → Confluence 기록 + 팀 재편성 검토 + 채널에서 자연스럽게 페이드아웃

## 10. 운영진 셀프 점검 (매주 금요일)

```
[ ] 이번 주 수요일 공유 빠짐없이 진행되었나?
[ ] 미제출자 1:1 케어 했는가?
```

> 참가자에게 "나도 할 수 있겠다"는 가장 큰 신호가 됩니다.
